The standard form of a nonlinear constrained problem is:

\[\begin{aligned} & \min\_x f(x)\\\\ \text{s. t.} \ \ & h(x) = 0\\\\ & g(x) \\le 0\\\\ \end{aligned}\]

Any optimisation problem can be written in this form. We will discuss various cases.

  1. \(g(x) \geq 0\) can be converted to \(-g(x) \leq 0\)

  2. \(g(x) < 0\) might look different. But it is essentially the same as \(g(x) \leq 0\) since all the numerical algorithms work in tolerances for constrain satisfaction. If we wish ensure that this constrain is satisfied to certain tolerance, then we can pose if as \(g(x) + \epsilon \leq 0\) where \(\epsilon\) can be set to the desired value.

  3. \(g(x) > 0\) can be posed as \(-(g(x) + \epsilon) \leq 0\)

  4. \(h(x) \neq 0\)

I have personally never encountered this kind of a constraint. But I could set \(- g(x)^2 \leq 0\). After optimisation, you will check if the constraint is active. If it is active, then you can perturb the optimum little bit. Strictly speaking, as all optimisation algorithms satisfy constraints within a tolerance, we do not really have g(x)=0 even if the constraint is active. If you want to make sure that \(| g(x) | > \epsilon\), where \(\epsilon\) is some distance that you want to impose from \(0\) then I could set

\(g(x) + \epsilon \geq 0\) and \(g(x) - \epsilon \ leq 0\)
